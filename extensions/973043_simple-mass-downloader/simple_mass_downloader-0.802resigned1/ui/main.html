Default name mask is {name}.{ext} {auto}, this is usually equivalent with {auto:name}.{auto:ext}
Other masks: {name}, {text}, {url}, {title}, {alt}, {host}, {ext}.
Date masks: {yyyy}, {yy}, {MM}, {dd}, {hh}, {mm}, {ss}, {ms}.
Counter masks: {counter} and {cc...c}. For example, {cccc} will generate the sequence 0001, 0002...etc.
URL parts masks: {url:n}. If url = http://example.com/aaa/bbb/ccc.jpg then {url:0} = example.com, {url:1} = aaa and so on. Also {url:-1} = ccc.jpg, {url:-2} = bbb etc.
A more advanced type of mask use the URL search parameters (if available). For example, if the search part of an url is foo=1&bar=2 we can define a mask as {search:foo} or {search:bar}
The literals { and } as such must be escaped: \{, \}

Basic example: term_1 term_2. This shows those resources for which the current displayed attribute contains term_1 and term_2.

But it's also possible to use here the special syntax source:term where source is all|url|name|text|title|alt (or shorter corresponding variants a|u|n|x|t|l). Negative forms are also possible (using "-" operator).

Filtering examples:

  • all:term_1 shows those resources for which at least an attribute contains term_1. Shorter version a:term_1.
  • url:term_1 text:term_2 shows those resources that have term_1 somewhere in their url and term_2 somewhere in their link text. Shorter version: u:term_1 x:term_2.
  • url:term_1 -text:term_2 shows those resources that have term_1 somewhere in their url and not term_2 in their link text. Shorter version: u:term_1 -x:term_2 (note that the search terms do not have spaces).

Set a pattern URL and import the generated URLs
For example http://example.com/image-[001:100].jpg

These pattern URLs use the special batch descriptors [start:end] or [start:end:step] where

  • start/end are positive integers or ascii letters
  • [optional] step is an integer (eventually negative)

Note that start/end must have the same type integer|upper|lower.
More examples:
Batch descriptor       Generated result
[1:100] 1, 2, 3 ... 100
[1:100:3] 1, 4, 7 ... 100
[001:100] 001, 002, ... 099, 100
[100:1:-3] 100, 97, 94 ... 1
[a:z] a, b, c ... z
[A:G:2] A, C, E, G
