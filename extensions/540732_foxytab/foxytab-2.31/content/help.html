Preamble

Actions works from any Tab, active or background. Firefox restricts some functions (e.g. copy/prompt/warning) on some URLs.

How to Use

Editing is non-destructive. Click Save to apply the new settings.

Tab, Highlighted, All, Left, & Right

  • Tab actions relate to the right-clicked tab
  • Highlighted tabs takes priority in single tab actions which means if there are highlighted tabs (besides the active tab), the action will apply to the highlighted tabs
  • All actions relate to the every tab in the active window
  • Left/Right actions relate to the tab to the left or right of the right-clicked tab

Preferences: Import/Export

You can import/export Preferences (for backup or share) from/to a local file on your computer.

Import is non-destructive. Click Save to apply the new settings.

Prior to 2.16, there was only one SAVE and clicking it, saved the entire data.

Since 2.16, each section has its own SAVE which improves the performance as there are many checks in each section and some require the data to be built before saving.

The side-effect of above is that when importing, the SAVE in the Options only saves the Options, and not the other sections. Their SAVE have to be clicked one by one as well (or ignored if desired in some cases).

Theme

Create, Import, Export & Apply your own Theme

Theme is made up of a "header" graphic image file, which skins the default Firefox UI background.

Creating a Theme Image

A foreground image to be added to the header area and anchored to the upper right corner of the header area.

Image Requirements:

  • Images should be 200 pixels high to ensure they always fill the header space vertically.
  • Dimensions should be 3000px wide × 200px high
  • PNG, JPG, APNG, SVG, GIF (animated GIF isn’t supported) or Data:URI file format
  • Image must not be larger than 300 KB in file size

Tips

  • Subtle, soft contrast images and gradients work best; highly detailed images will compete with the browser UI.
  • Firefox may reveal more of the lower portion of the image if another toolbar or other UI element is added to the top of the window.
  • The upper right-hand side of the image should have the most important information—as a user increases the width of the browser window, the browser reveals more of the left-hand side of the image.
  • Ultrawide monitors can run to 3440px wide, to look good on these monitors, either increase the size of the image or ensure the left of the image fades to a solid colour.

Import

Import ready FoxyTab Themes (JSON files), or import an image into the theme that you are making.

Imported Image will be converted to Data:URI entered in "Header Image" section, if it is empty and if not, in the "Additional Backgrounds" section.

FoxyTab Theme manifests can be imported into FoxyTab in the following JSON format, using any available properties listed on MDN theme.
The main difference is that images must be in Data:URI format. Maximum size allowed for the JSON is set to 500kb.

The minimum requirements for a theme are "accentcolor" & "textcolor".

Example 
{
  "name": "Theme Name",
  "description": "A short explanation of your theme",
  "version": "1.0",
  "homepage_url": "https://.../",
  "author": "your name",

  "theme": {
     "images": {
       "theme_frame": "data:image/jpg;base64 ...",
       "additional_backgrounds": [ "data:image/png;base64 ..."]
     },
     "colors": {
       "frame": "green",
       "tab_background_text": "#000"
     },
     "properties": {
       "additional_backgrounds_alignment": [ "top" ],
       "additional_backgrounds_tiling": [ "repeat"  ]
     }
   }
}

Redirect

Setup Tab Redirect rules to forward tab URLs. one rule per line, with 2-3 parts separated by space/s. Matching is case-insensitive.

Format

match redirect (replace) exclude-match (optional)

The entire URL is matched in match/exclude-match.

Example 
http://example.com/*                            https://example.com/$1
https://*.example2.com/*                        https://www.example2.com/$1           https://my.example2.com/*
https://([^.]+)\.example3\.co\.uk/(\d+)/(\w+)   https://$1.example3.com/$2/$3

Wildcard

Wildcard implementation is similar to Match Patterns

Examples
Pattern match no-match
*://*.example.org/* http://example.org/
https://www.example.org/
http://www.sub.example.org/aaa/ 		http://www.example.com/
*://example.org/ http://example.org/
https://example.org/ 		https://www.example.org/
http://example.org/aaa/
https://*/path https://www.example.org/path http://example.org/
http://example.org/path

Regular Expression

Regular Expressions require escaping and therefore any match/exclude-match that includes backslash \ would be regraded as Regular Expression.

Regular Expression string (without start & end slashes and flags) uses new RegExp() (case-insensitive) therefore (single) escaping especial characters are necessary but not needed for forward slashes /.

Replacement Parameter

$1, $2, $3, ... can be used in redirect to replace * in wildcard type, or (..) in regular expression type matches.
