Integration

An additional step is needed to start the JDownloader instance and send links to it. You must install the minimal native client. This client enables communication between the extension and the download manager.

  • Click here to download the ZIP package for your operating system or click here to browse all available download options.
  • Extract the downloaded package into a local directory
  • Now open a terminal in this directory (the directory that contains install.sh script) and run ./install.sh. You can alternatively drop the install.sh file to a terminal window and press the Enter key!
  • Now double-click the install.bat file

Native integration works!

If you still get this page, the native side is not responding. Most likely:
  • The extension cannot find the native application. Open the options page and set the correct path
  • The native application is found but exited with a non successful exit code
  • You must have the JDownloader already installed to use this extension. This extension only integrates your browser with the existing manager.
  • The installer script copies some files to two locations to enable your browser to detect the native client. If you have NodeJS already installed in your system, the installed version will be used. Therefore, it is recommended to have an updated version of NodeJS. In case you don't have this application, the included version will be used instead. Read more here.
  • To completely remove this native client, run ./uninstall.shuninstall.bat.
  • To check the native client's accessibility, click on the "Check Connection" button. If you still get this page even when the "Check Button" reports successful connection, it means the application cannot be accessed (application's path is not correct in the options page). If the path to the executable is correct and the native client is connected, you can debug the connection by enabling console logs.
  • The native client can operate without requiring administration permission.
Main controlling Options: You can control the running instance of JDownloader by using a second terminal window. You can enter pre-defined commands in this window using the same executable used to start the application. 
JDownloader -pause
JDownloader -continue
JDownloader -stop
JDownloader -start
JDownloader -update
JDownloader -speed
JDownloader -maxcon <maxNumConn>
JDownloader -speed <speed>
JDownloader -maxDls <maxDls>
JDownloader -addwatch <folderLocation>
JDownloader -remwatch <folderLocation>
JDownloader -add <link>
JDownloader -remove <packageName>
JDownloader -order <packageName> <newPosistionInQueue>
JDownloader -queueFirst <packageName>
JDownloader -queueLast <packageName>
JDownloader -force <packageName>
JDownloader -quit
Reporting: Additional options, such as reporting, can also be integrated into your command. 
JDownloader -status queue
JDownloader -status downloads
JDownloader -status all
JDownloader -status watchfolder
