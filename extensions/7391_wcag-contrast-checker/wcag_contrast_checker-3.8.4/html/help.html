WCAG Contrast Checker is a browser extension that analyses the color contrast between foreground and background of all elements in the page.

The tool can also analyse the color contrast of documents included with iframes. For accessing to their results, it includes a document selector that switches between all the HTML documents.

The extension does not analyse the color of the images. If the element is an image, the detected color for the foreground is the one that its alternative text would use, and if the element includes and image as background, the background color is the one that it has defined without considering the image.

Visible and Hidden elements

The tool identifies the elements that use common techniques to hide content (for collapsible elements, fly-outs, menus, etc.). Those elements are not evaluated.

Color blindness selector

By choosing one of the color blindness options, the colors in the page will emulate that visual condition and the color contrast is reevaluated according to it.

Auto-refresh (refresh on DOM updates)

The analysis can be performed to be executed on every DOM modification (additions or removal of DOM elements, as well as on changes of class values). Although this feature gives better feedback when the page updates content without reload, it can also impact in the general performance.

Manual color analysis

The extension includes a color analyser that allows to introduce a color combination to evaluate its contrast.

Color contrast in WCAG (Web Content Accessibility Guidelines)

WCAG define two levels of compliance that differ only in the contrast ratio:

  • Level AA:
    • 4.5:1 for normal text
    • 3:1 for large text
    • 3:1 for graphics and user interface components (such as form input borders)
  • Level AAA:
    • 7:1 for normal text
    • 4.5:1 for large text

Large text is defined as 14 point (typically 18.66px, or 1.2rem when the root font is 16px) and bold or larger, or 18 point (typically 24px or 1.5rem) or larger.
