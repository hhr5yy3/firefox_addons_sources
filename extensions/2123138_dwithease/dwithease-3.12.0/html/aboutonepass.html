Now you can leverage additional security when storing your accounts in DWithEase

The OnePass can encrypt all other accounts using a single pass phrase

Like a lot of password managers, we have the following approach:

  • Zero-knowledge policy - after password encryption with AES-256 bit encryption with PBKDF2 SHA-256 and salted hashes no one has access to the passwords unless he has the OnePass
  • OnePass - this is a pass phrase entered on occasion to unlock the vault and gain access to all stored passwords
  • Limited time user sessions - the OnePass unlocks the vault for a limited period of time (e.g. 30 minutes), after that period the vault is locked and the user is prompted for the OnePass again

We now use PBKDF2 to encrypt the passwords opened with the OnePass.

The aim is to provide additional security that will reassure you of the safety practices we employ.

You can also, if needed, remove the OnePass if you choose to stop using this feature.

All passwords stored in the extensions are accessible only within an opened session ranging between [ 15-30 mins, 1-2-4 hours, or until browser closed ] during which the password will be stored in the background execution context. You can lock the OnePass at any time, using the padlock in the pop-up window.
