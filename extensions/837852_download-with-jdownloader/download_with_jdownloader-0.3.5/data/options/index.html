The following are the supported keywords to pass to JDownloader: [URL] (link URL or the source of an image, video, or audio file), [REFERRER] (URL of the page that contains the link), [USERAGENT] (user-agent string of the browser), [FILENAME] (name of the file detected by your browser; if no name was detected, an empty string is used), [DISK] (absolute path to the detected file on the user's disk; if no file was detected, an empty string is used), [COOKIES] (absolute path to a text file that contains relevant cookies).
When sending cookies from a hostname to JDownloader, be aware that this information may contain private details such as personal tokens or credentials. Use this feature with caution.
If you leave this box unchecked, the extension will instruct JDownloader to keep the jobs in the link grabber section instead of starting the downloads immediately.
When sending multiple links to an external download manager, the extension can either send them all at once or one by one. Choose this option if the download manager accepts multiple links in a single command. Otherwise, the extension will send each link individually with a delay.
When sending multiple jobs to JDownloader and the "Batch mode" is not enabled, there will be a delay between requests.

Interrupt

To interrupt the built-in download manager for specific hostnames, enable the extension's interruption mode from the browser action button and enter a comma-separated list of hostnames. Downloads from these specified hostnames will be automatically interrupted. For all other hostnames, you'll need to use the right-click context menu to send the job to jDownloader. Ensure that the hostname you enter matches the actual downloadable link, not the parent page.

Documentation

To control the running instance of JDownloader, open a second terminal window and enter predefined commands using the same executable that started the application. These commands will enable you to manage the program. 
JDownloader -pause
JDownloader -continue
JDownloader -stop
JDownloader -start
JDownloader -update
JDownloader -speed
JDownloader -maxcon <maxNumConn>
JDownloader -speed <speed>
JDownloader -maxDls <maxDls>
JDownloader -addwatch <folderLocation>
JDownloader -remwatch <folderLocation>
JDownloader -add <link>
JDownloader -remove <packageName>
JDownloader -order <packageName> <newPosistionInQueue>
JDownloader -queueFirst <packageName>
JDownloader -queueLast <packageName>
JDownloader -force <packageName>
JDownloader -quit
Additional features like reporting can be integrated into your command. 
JDownloader -status queue
JDownloader -status downloads
JDownloader -status all
JDownloader -status watchfolder
