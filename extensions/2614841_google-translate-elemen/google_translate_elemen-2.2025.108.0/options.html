Google Translate Element





Automatically inserts Google Translate Element into the page if the language of the page is different from these languages. Even if the language of the page is the same to one of these languages it can be inserted manually from the context menu. When you click the default button, language information is acquired from about:preferences → Language.
Example: en, de, zh. Separated by commas.
Reference: Language support Google Cloud (different service from Google Translate Element), List of ISO 639-1 codes - Wikipedia.

Separate with commas without spaces. Example: en,de,zh-CN.
Reference: Language support Google Cloud (different service from Google Translate Element), List of ISO 639-1 codes - Wikipedia, ISO 3166-1 - Wikipedia.
If you empty the list, all languages are displayed in the translation destination language of Google Translate Element. The default value is obtained from about:preferences → Language, but it may not be useable as it is. Because some languages require region codes like zh-CN or zh-TW, and in other languages, if you add region codes like en-US it will not work and you need to delete them to make it like en.


(Regular expression)

Example 
^https://www\.google\.com/search\?
^https://.*\.wikipedia\.org/wiki/

Escape the regular expression special characters such as ".", "*", "?" with a backslash. Add "^" at the beginning of the pattern to match the beginning of URLs.

Example

}

To get better translation results, remove these tags from the web page and convert the content texts to simple texts. If HTML tags are included in the middle of a sentence, the sentence passed to the translation engine may be different from a simple sentence without tags in order to reflect those tags in the translated sentence, which sometimes results in strange translation results. Removing these HTML tags may give you more correct translation results. However, it becomes difficult to read because there are no links or decorations. To remove the tag, execute the process from the context menu, shortcut key, etc., before Google Translate Element translates the page (when the original text is displayed). If you want to restore the contents of the page, you need to reload the page. When adding tags, separate them with white spaces or line breaks.

Google Translate Element may not work on some pages due to security restrictions on those pages. In this case, a message is displayed in the Browser Console. If it does not work, you can translate that page in a new tab from the context menu as an alternative.

If you disable the add-on from the context menu, Google Translate Element will not be inserted automatically. Again, you can insert it manually from the context menu.

'@url@' is replaced with the URL of the page. '@text@' is replaced with the selected text. '@lang@' is replaced with "Default Target Language". The value of Default Target Language is obtained from about:preferences → Language. However, since it may be different from the format of the target language of each translation site, it may not be used as it is. Some languages may require region codes such as 'zh-CN' and 'zh-TW', and in other languages, it may not work if the region code is attached like 'en-US' or 'pt-BR', so it may need to be changed to 'en' or 'pt'. In that case, you must manually replace default target language with the appropriate language code. '@source_lang@' is replaced with the language code of the selected string or the language code of the page. However, this does not always match the expression of the language code used by each translation site.

The value of '@lang@' in this browser:

A character string that matches the last capture group in the pattern described in "Regular expression for extracting target language" is extracted from the URL of the translation result page as the target language for translation. When the language of the translation destination is changed interactively on the translation result page, if the URL changes accordingly, the language is extracted and stored. In the next translation, '@lang@' will be replaced with this value instead of the default target language before translation. If you always want to translate to the default language, leave the regular expression empty. This function cannot be used on translation sites where the URL does not change even if the target language is changed interactively.

When the language of the page or the language of the selected text matches the "Languages not to be automatically inserted", translation is performed at the second translation site. This can be used, for example, to translate native language sentences into other languages. If you hold down a modifier key ("Alt", "Command", "Ctrl", "MacCtrl" or "Shift") and click on the context menu or use a shortcut key, you can use the second translation site to translate whatever language the page or selected text is. If you register another site in the second translation site, you can compare the two translation results.

Example

omnibox

Translate text by entering commands and text in omnibox (address bar, location bar). It is convenient to use the keyboard shortcut to move the focus to omnibox. F6, Alt + D or Ctrl + L on Windows.

Syntax: tr [-t target_lang] [-1|-2] [-d {nt|st|ct|w|sb|ba}] text

  • tr The trigger keyword to start translation with omnibox.
  • -t Specify the target language. If omitted, the default target language for the text translation site will be used. The target language format varies from translation site to translation site, so you need to know the appropriate language code for that site.
  • -1 Translate using the default text translation site.
  • -2 Translate using a second text translation site. If both -1 and -2 are omitted, the site used for translation is determined by the language of the entered text.
  • -d Specify the display destination of translation results.
    • nt New tab.
    • st Specific tab.
    • ct Current tab.
    • w Window.
    • sb Sidebar. The add-on cannot automatically open the sidebar, so the user must open it manually.
    • ba Browser action panel. The add-on cannot automatically open the browser action panel, so the user must open it manually.
    • pa Page action panel. The add-on cannot automatically open the page action panel, so the user must open it manually.
  • text The sentence to translate.

Examples: tr Hello, world. --- Use the default translation site to translate to your native language. tr -2 -t de -d sb みなさんこんにちは。 --- Translate to German using a second translation site and display the result in the sidebar.

Specific tab

If the specific tab exists, reuse that tab and display the translation result on that tab without opening a new tab.



Memorize the information of the translation result display window and a specific tab across sessions, and reuse the same tab and window after restarting the browser.

Translation result window

The position and size of the window displaying the translation result are stored when the focus is transferred to another window of the browser. The first and second page translation sites and the text translation site are stored separately.


Sidebar

Clipboard


The contents of the clipboard can be translated by using shortcut keys, clicking an item in the context menu with the second mouse button, or holding down the control key and clicking a button in the browser action panel or page action panel.

Shortcut keys

To set shortcut keys, go to about:addons -> ⚙️ Gear icon -> Manage Extension Shortcuts.
