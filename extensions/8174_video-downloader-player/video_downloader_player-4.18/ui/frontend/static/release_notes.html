Versions 4.16, 4.17, 4.18 release notes

Version 4.18

External pages are embedded into the extension, and all links to Native app are removed
Ant Video downloader is discontinued since March 1, 2023.

Version 4.17

This release includes video detection fix for web-site netu.tv/netu.io

Please note that web-site netu.tv/netu.io is very aggressive against debugging, 3rd party video downloaders and contains geo-restricted / VPN-restricted content. This fix may be temporary. Let us know in the support ASAP if you have any issues downloading videos from the web-site again.


Version 4.16

This release includes support of 32-bit architecture, new features and bugfixes

Support of 32-bit architecture
If you're using 32-bit version of Windows operating system, you can use Ant Video downloader without downloading 32-bit version of video converter. Native app installer now contains both 32- and 64-bit versions of ffmpeg and ffprobe tools. Native app will use 32- or 64-bit version of video converter automatically, depending on your system's type.

Take into account that 32-bit version of video converter is lower than 64-bit, and thus it may have support less number of websites comparing to 64-bit. We advise you to upgrade your system to 64-bit if possible.

New setting for private downloads location
Version 4.16 of Ant Video downloader now supports custom location for private downloads. The Settings page now has new section Downloads—Private mode. In the section, there are 3 radio buttons that control the new setting.

Ask each time for downloads location — enabled by default, AVD will act as before, asking each time for download location and popping up Save as dialog window.

Download in the same Downloads folder used by the normal mode — will download all private mode videos into the same Downloads folder specified in Downloads—General

Download into the following location — will enable Browse button and allow setting up of custom folder for private mode downloads.

NOTE: if you choose this option and won't set up custom folder location, private mode downloads will fail with error message saying you did not set up folder location.

YouTube: added drop-down for containers, ability to preselect best quality
YouTube's videos now have another select to filter qualities by container.

Pre-select best quality video setting is now applied to videos filtered by the container.

YouTube: download audio only
When you select Download audio only in the containers select, you can download audio streams only, without video. YouTube audio downloads are carried out by the web browser, Native app is not required.

Support of previous per-browser downloads
If you start video download via browser, and then reload Ant Video downloader or restart the web browser, download will appear as ongoing and won't move to previous downloads.

This feature is applied only to via browser downloads, and not usual downloads which are carried out by the Native app. This is because web browser's native messaging API doesn't support re-establishing of connection to an existing native app instance.

Bugfixes
• Bugfix: Vimeo: some videos are downloaded with the same quality, no matter what is selected in Quick Select
• Bugfix: MPEG-DASH, Vimeo, YouTube Now Playing: audio stream is not preselected when Pre-select best quality setting is turned off
• Various small UI fixes
