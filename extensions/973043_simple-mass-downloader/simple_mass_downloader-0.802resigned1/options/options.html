First step is to define a new folder or open a previously defined folder

Every automatic folder has (at least) an associated rule set. A resource will be saved in this folder only if it satisfies all rules from this rule set (if there are more rule sets, it's enough to satisfy all rules from one of his rule sets). An individual rule must be written in the form

keyword: word1, word2, ...

where keyword may be ext|url|name|host|text|alt|title, for example ext: jpg, png, bmp. So a resource satisfies this sample rule only if its file extension is jpg or png or bmp.

In this dialog, the user may define new folders or select for editing previous folders. A new folder is written in the form aaa/bbb/.../zzz. Please note that every folder is defined relative to the default download directory (this is a web extensions security limitation). So a folder defined as John/Music is, in fact, a folder named Music located inside a folder named John, and this last one is created inside the browser's default download directory.

The folder name can include the following masks: {yyyy}, {yy}, {MM}, {ww}, {dd}, {host}, {url:n}, {ext}

A folder may be also removed (but this only remove it from automatic folders list, not delete it from disk)

Write here the favorite folders (relative to the default download directory), one folder per line. For example Music or Docs/Math/Algebra, with optional masks: {yyyy}, {yy}, {MM}, {ww}, {dd}, {host}, {url:n}, {ext}
There are actually these experimental features:
  • Extra scan
  • Throttle mode
See extension's page help for further info
Default name mask is {url}{newline} (meaning one URL per line)

Other masks: {name}, {text} {url}, {title}, {alt}, {host}, {ext}, {newline}, {tab}.

Date masks are also available: {yyyy}, {yy}, {MM}, {dd}, {hh}, {mm}, {ss}, {ms}.

It's also possible to use the counter masks {counter} and {cc...c}. For example, {cccc} will generate the sequence 0001, 0002...etc.

The literals { and } as such must be escaped: \{, \}
The resource list items may be displayed with appropriate file icons if these are known.
The web extensions system allows file icons to be retrieved only after a file was downloaded. So this button initiate a series of empty file downloads with various common mime types with the purpose of building and store a list of icons for popular file types (these icon sets vary from one computer to another)
